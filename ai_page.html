What is Generative AI?

Generative AI (GenAI) refers to AI systems that can create new content — such as text, code, or images — based on patterns learned from large amounts of data.

Unlike traditional programs that follow fixed rules, GenAI models like ChatGPT or GitHub Copilot learn from millions of examples and can generate human-like responses and working code.

💡 Think of it as a very experienced coding mentor who has read every Stack Overflow post and GitHub repository — and can instantly suggest solutions to your problems.

Traditional Programming vs. Generative AI

Traditional ProgrammingGenerative AI

Example: AI Helping with JavaScript

You ask an AI to help write a function for a webpage you are building:

Your Prompt
Write a JavaScript function that takes an array of numbers and returns only the even numbers.
AI Response
// Filters and returns only even numbers from an array function getEvenNumbers(numbers) { return numbers.filter(num => num % 2 === 0); } const result = getEvenNumbers([1, 2, 3, 4, 5, 6]); console.log(result); // [2, 4, 6]
✅ AI gives you a working solution — but it's your job to understand what it does and test it in your project.

Where Does GenAI Fit in Your Workflow?

  • Starting a project — Generate boilerplate HTML/CSS or an Express server setup
  • Debugging — Paste your error message and ask AI to explain it
  • Learning — Ask AI to explain unfamiliar code or concepts
  • Refactoring — Ask AI to rewrite messy code more cleanly
  • Testing — Generate test cases for your functions

Prompting Techniques

Learn how to write clear, effective prompts to get the most useful code and explanations from AI.

2of 4

Why Does Prompting Matter?

AI tools are only as good as the instructions you give them. A vague prompt gives a vague answer. A clear, detailed prompt gives a precise, useful answer.

⚠️ Garbage in, garbage out. If your prompt is unclear, the AI will guess — and may give you something that looks right but does not work for your situation.

Technique 1: Be Specific and Give Context

Always tell the AI what you want, why you need it, and what environment you're working in.

Vague Prompt
Make a button.
Result
A generic button with no styling or functionality — probably not what you wanted.
Specific Prompt
I'm building a student registration form in HTML and CSS. Create a "Submit" button that is blue, rounded, full-width, and shows a hover effect. No JavaScript needed.
AI Response
.submit-btn { background-color: #2563eb; color: white; width: 100%; padding: 12px; border: none; border-radius: 8px; cursor: pointer; transition: background 0.2s; } .submit-btn:hover { background-color: #1d4ed8; }

Technique 2: Zero-Shot vs. Few-Shot Prompting

Zero-shot — No examples, just a direct instruction. Good for simple tasks.

Few-shot — You provide 1–2 examples. Ideal when you have a specific pattern in mind.

Few-Shot Example
Convert these variable names from camelCase to snake_case. Input: firstName → Output: first_name Input: totalScore → Output: total_score Input: userEmailAddress → Output: ?
AI Response
Output: user_email_address
💡 Few-shot prompting works great for repetitive code — like multiple React components or database queries that follow the same pattern.

Technique 3: Role Prompting

Tell the AI to act as a specific expert. This shifts the style and depth of its response.

Role Prompt
You are a senior Node.js developer teaching a first-year polytechnic student. Explain what middleware is in Express.js using a simple analogy, then show a basic example.
AI Response
// Middleware is like a security guard at a mall — // every visitor (request) passes through before reaching the shops (routes). const app = require('express')(); app.use((req, res, next) => { console.log(`${req.method} ${req.url}`); next(); }); app.get('/', (req, res) => res.send('Hello!')); app.listen(3000);

Technique 4: Chain-of-Thought

Ask AI to reason step by step before giving a final answer. This reduces mistakes on complex problems.

Chain-of-Thought Prompt
I want to fetch data from an API and display it in a React component. Think through the steps first, then write the code using fetch() with useState and useEffect hooks.

Prompting Cheat Sheet

TechniqueWhen to UseExample Phrase
Be SpecificAlways"I am building X using Y. Write Z that does..."
Zero-ShotSimple, clear tasks"Write a function that..."
Few-ShotRepeating patterns"Here are 2 examples — now do the same for..."

Code Review & Analysis Using AI

Use AI to spot bugs, improve code quality, understand unfamiliar code, and refactor effectively.

3of 4

What is Code Review?

Code review is the process of examining code to find bugs, improve readability, and ensure it follows good practices. Traditionally done by a senior developer — now AI can assist too.

ℹ️ AI will not replace human code review, but it can catch common mistakes quickly and help you learn why something is wrong — not just that it is wrong.

1. Debugging with AI

When you get an error, paste your code and the error message into AI chat. Ask it to explain what's wrong and suggest a fix.

Student Prompt
This Express code gives me: "Cannot set headers after they are sent to the client": app.get('/user', (req, res) => { if (!req.query.id) { res.send('No ID provided'); } res.send('User found'); });
AI Explanation
// Problem: res.send() is called twice. // Fix: add return so execution stops after the first response. app.get('/user', (req, res) => { if (!req.query.id) { return res.send('No ID provided'); // ← return added } res.send('User found'); });

2. Asking AI to Review Your Code

Code Review Prompt
Review this React component for bugs or bad practices. I am a beginner building a to-do list: function TodoList({ items }) { return ( <ul> {items.map(item => ( <li>{item.text}</li> ))} </ul> ); }
AI Feedback
// Issue: Missing 'key' prop. React needs this to track list items. function TodoList({ items }) { return ( <ul> {items.map(item => ( <li key={item.id}>{item.text}</li> // ← key added ))} </ul> ); }

3. Understanding Unfamiliar Code

Explanation Prompt
Explain this JavaScript line by line for a first-year student: const doubled = [1,2,3].map(x => x * 2);
AI Response
// [1,2,3] → an array of three numbers // .map() → applies a function to each item // x => x*2 → multiply each number by 2 // Result: doubled = [2, 4, 6]

4. Refactoring Code with AI

Refactor Prompt
Refactor this into cleaner code without changing functionality: function c(a,b,d){if(d==="add"){return a+b;}else if(d==="sub"){return a-b;}else{return 0;}}
Refactored Code
function calculate(numA, numB, operation) { if (operation === 'add') return numA + numB; if (operation === 'sub') return numA - numB; return 0; }

Best Practices

  • Always read and understand AI suggestions before applying them.
  • Test the fixed code — AI can introduce new bugs while fixing old ones.
  • Ask AI to explain why something is wrong, not just how to fix it.
  • Use AI as a learning tool — understand the fix so you don't repeat the mistake.
  • Never blindly copy-paste fixes without understanding them.

AI Ethics & Limitations

Understand the risks of AI-generated code, academic integrity, and how to use AI responsibly.

4of 4

Limitation 1: AI Hallucinations

AI models can confidently produce code that looks correct but is completely wrong. This is called a hallucination.

🚨 AI may invent function names or library features that do not exist. Always verify AI-generated code by running it and checking the official documentation.
Example of a Hallucination
app.useAutoAuth({ provider: 'google' }); // This method does NOT exist in Express.js. // AI invented it — your code will crash.

Limitation 2: Outdated Information

AI is trained on data up to a certain date and may not know about the latest versions, new APIs, or recent security patches.

ScenarioRiskWhat To Do
AI suggests a deprecated React methodCode breaks in newer versionsCheck the official React docs
AI recommends an outdated npm packageSecurity vulnerabilitiesRun npm audit after installing
AI shows old Express syntaxIncompatibility with Express 5+Verify against the official Express docs

Limitation 3: No Knowledge of Your Full Project

AI only sees what you paste into the chat. It has no knowledge of your full project structure, existing variables, or other files — which can lead to suggestions that break other parts of your code.

💡 Describe your project structure or paste related files so the AI can give more accurate, context-aware advice.

Academic Integrity

✅ Acceptable Use❌ Not Acceptable
Using AI to understand a conceptSubmitting AI-generated code as entirely your own
Using AI to debug your own codeHaving AI complete entire assignments for you
Using AI for a first draft, then rewriting itCopy-pasting AI output without understanding it
Declaring AI use when required by your lecturerNot disclosing AI use when required
⚠️ Always follow your school's AI policy. When in doubt, ask your lecturer before using AI on a graded task.

Intellectual Property and Privacy

  • Never paste private data — Credentials, API keys, and user data should never go into a public AI tool.
  • Code ownership is unclear — AI-generated code may reflect copyrighted patterns. Check your organisation's policy for production projects.
  • Your prompts may be stored — Be careful what confidential project details you share.

AI as a Tool, Not a Crutch

🎯 The goal is to become a better developer — not just a better AI user. Use AI to speed up your learning and remove blockers, but make sure you understand everything you submit. Your skills are what will matter in your career.
